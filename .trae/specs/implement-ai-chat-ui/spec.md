# AI对话应用UI设计实现 Spec

## Why
为Operit应用实现完整的AI对话界面，提供直观的用户交互体验，包括导航、聊天、模型配置、功能菜单和侧边栏等核心功能。

## What Changes
- 实现顶部导航栏（应用导航栏 + 子导航栏）
- 实现主聊天内容区域（支持历史对话滚动）
- 实现底部输入交互区域（输入框 + 功能栏）
- 实现模型配置弹窗
- 实现加号功能菜单
- 实现更多设置面板
- 实现整合侧边栏（用户信息 + 功能菜单）

## Impact
- Affected specs: 新增UI界面能力
- Affected code: UI组件、状态管理、交互逻辑

## ADDED Requirements
### Requirement: 顶部导航栏
系统应提供顶部导航栏，包括：
- 左侧：汉堡菜单按钮（展开/收起全局侧边栏）
- 中间：页面标题 "AI 对话 - 新对话 [时间]"
- 右侧：窗口图标、左右箭头导航按钮（切换历史对话）

### Requirement: 子导航栏
系统应提供子导航栏，包括：
- 左侧：刷新图标、方形图标（收起/展开侧边栏）、AI助手头像及名称 "Operit"
- 右侧：tokens统计区域（如 "14.50k"）、用户头像图标（点击弹出侧边栏）

### Requirement: 主聊天内容区域
系统应提供主聊天内容区域，包括：
- 默认显示欢迎语 "周末愉快，工作的事交给我"
- 支持展示用户与AI的对话内容
- 支持上下滚动查看历史对话

### Requirement: 底部输入交互区域
系统应提供底部输入交互区域，包括：
- 输入框：提示文字 "请输入您的问题..."
- 输入框右侧：全屏图标（切换聊天区域全屏）
- 输入框下方功能栏：
  - 左侧："模型配置" 下拉按钮
  - 中间：设置图标
  - 中间偏右：加号按钮（弹出功能菜单）
  - 最右侧：麦克风按钮（语音输入）

### Requirement: 模型配置弹窗
系统应提供模型配置弹窗，包括：
- 思考状态：显示 "思考: off"
- 思考模式：开关按钮（默认关闭）
- 思考引导：开关按钮（默认关闭）
- Max 模式：开关按钮（默认关闭）
- 模型选择：显示 "模型：未选择"
- 底部操作按钮："默认配置"、"管理配置"

#### Scenario: 打开模型配置弹窗
- **WHEN** 用户点击 "模型配置" 按钮
- **THEN** 弹出模型配置悬浮面板

#### Scenario: 应用模型配置
- **WHEN** 用户在模型配置弹窗中选择模型
- **THEN** 弹窗自动关闭并应用所选模型设置

### Requirement: 加号功能菜单
系统应提供加号功能菜单，包括：
- 照片：选择本地照片上传
- 拍照：调用相机拍照并上传
- 记忆：调用AI记忆功能
- 音频：上传音频文件或录制
- 文件：上传本地文件
- 屏幕内容：屏幕截图并上传
- 当前通知：获取系统通知并上传
- 当前位置：获取地理位置并上传

#### Scenario: 打开功能菜单
- **WHEN** 用户点击加号按钮
- **THEN** 弹出垂直悬浮功能菜单

### Requirement: 更多设置面板
系统应提供更多设置面板，包括：
- 记忆选项：
  - 记忆模式：下拉切换（默认 "Default"）
  - 记忆附着：开关按钮（默认开启）
  - 手动更新记忆：开关按钮（默认关闭）
- 功能开关：
  - 自动朗读：开关按钮（默认关闭）
  - 自动批准：开关按钮（默认关闭）
- 禁用项设置：
  - 显示 "禁用项: 0/4"
  - 禁用流式输出：开关按钮（默认关闭）
  - 禁用工具：开关按钮（默认关闭）
  - 禁用户偏好描述：开关按钮（默认关闭）
  - 禁用LaTeX描述：开关按钮（默认关闭）
- 底部操作按钮："工具管理"

#### Scenario: 打开更多设置面板
- **WHEN** 用户点击设置图标
- **THEN** 弹出设置悬浮面板

### Requirement: 整合侧边栏
系统应提供整合侧边栏，包括：

#### 用户信息区域
- 顶部：用户头像、用户名
- 账户信息：显示 "个人免费版"，"升级" 按钮，"暂无订阅计划"
- 积分：显示 "积分 14,431"（可点击进入详情页）
- 功能入口：
  - 通用设置
  - 帮助中心

#### 功能菜单区域
- AI功能分类：
  - AI 对话
  - 助手配置
  - 包管理（当前选中，浅蓝色高亮）
  - 记忆库
- 工具分类：
  - 工具箱
  - 权限授予
  - 工作流
- 系统分类：
  - 设置
  - 使用手册
  - 关于
  - 更新历史

#### Scenario: 打开侧边栏
- **WHEN** 用户点击用户头像图标
- **THEN** 侧边栏从右侧滑出

#### Scenario: 关闭侧边栏
- **WHEN** 用户再次点击用户头像或点击侧边栏外区域
- **THEN** 侧边栏收起

### Requirement: 弹窗交互
系统应支持弹窗的打开和关闭：
- 点击弹窗外部区域可关闭弹窗
- 点击弹窗内的关闭按钮可关闭弹窗

### Requirement: 输入区域交互
系统应支持：
- 输入框支持文本输入
- 点击麦克风按钮切换为语音输入模式
